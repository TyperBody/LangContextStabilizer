# コンテキストスタビライザープラグイン

審査モデルを使用して会話コンテキストを検証・管理する LangBot プラグインです。

## 機能

- **コンテキスト審査**: 補助 LLM モデル（モデルB）を使用して会話コンテキストを審査
- **ステガノグラフィー検出**: インジェクション攻撃を示す可能性のある隠し文字やゼロ幅文字を検出
- **コンテキスト圧縮**: 重要な情報を保持しながら長いコンテキストを自動圧縮
- **プロンプトインジェクション保護**: 圧縮後に元のシステムプロンプトリマインダーを注入
- **適応型審査頻度**: 審査結果に基づいて審査頻度を自動調整
- **設定可能な頻度**: 審査頻度を設定（N回の会話ごとに審査）
- **タイムアウト保護**: 審査がメイン会話フローをブロックするのを防止

## インストール

1. プラグインをダウンロード
2. LangBot プラグインディレクトリに配置
3. LangBot Web インターフェースでプラグインを設定

## 設定

### 審査モデル設定

| 設定 | 説明 | デフォルト |
|------|------|-----------|
| audit_model_uuid | 審査に使用する LLM モデル | 必須 |
| audit_system_prompt | 審査モデルのシステムプロンプト | 内蔵 |

### 頻度設定

| 設定 | 説明 | デフォルト |
|------|------|-----------|
| audit_frequency | 審査間隔（ラウンド数） | 3 |
| enable_adaptive_frequency | 結果に基づいて頻度を自動調整 | false |
| frequency_increase_step | 失敗時の頻度増加ステップ | 1 |
| frequency_recovery_threshold | 頻度を下げるまでの連続成功回数 | 3 |
| min_audit_frequency | 最小審査間隔（最高頻度） | 1 |

### タイムアウト設定

| 設定 | 説明 | デフォルト |
|------|------|-----------|
| audit_timeout_seconds | 審査タイムアウト（秒） | 10 |
| timeout_action | タイムアウト時の処理 | remove_chunk |

### 圧縮設定

| 設定 | 説明 | デフォルト |
|------|------|-----------|
| max_context_length | 圧縮をトリガーする最大メッセージ数 | 50 |
| compress_target_length | 圧縮後の目標メッセージ数 | 10 |
| compression_model_uuid | 圧縮モデル（空=審査モデルを使用） | - |
| compression_prompt | コンテキスト要約のプロンプト | 内蔵 |
| enable_prompt_injection | 圧縮後に元のプロンプトを注入 | true |

### ステガノグラフィー検出

| 設定 | 説明 | デフォルト |
|------|------|-----------|
| enable_steganography_detection | ステガノグラフィー検出を有効化 | true |
| steganography_patterns | 検出用正規表現パターン | 内蔵 |

### 詳細設定

| 設定 | 説明 | デフォルト |
|------|------|-----------|
| chunk_size | 審査チャンクあたりのメッセージ数 | 5 |
| enable_logging | 詳細ログを有効化 | false |

## コマンド

| コマンド | 説明 |
|----------|------|
| `!ctxstab status` | 現在のセッション審査状態を表示 |
| `!ctxstab audit` | 次のメッセージで強制審査 |
| `!ctxstab compress` | 次のメッセージで強制圧縮 |
| `!ctxstab reset` | 審査カウンターをリセット |
| `!ctxstab config` | 現在の設定を表示 |

## 動作原理

1. **イベントリスニング**: `PromptPreProcessing` イベントをリスニングして会話コンテキストを取得
2. **頻度チェック**: 設定された審査頻度に基づいて審査するかどうかを決定
3. **ステガノグラフィー検出**: コンテキスト内のゼロ幅文字などを検出
4. **コンテキスト分割**: コンテキストを複数のチャンクに分割して審査
5. **モデル審査**: 審査モデルを呼び出して各チャンクが元の設定に準拠しているかチェック
6. **結果処理**: 審査結果に基づいてコンテキストを圧縮または削除
7. **適応型頻度**: 成功/失敗結果に基づいて審査頻度を調整（有効な場合）
